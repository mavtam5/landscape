üëã Introduction

Welcome to the EasyPeasy First Person Controller! A clean, modular, and extensible controller system built for Unity using the State Pattern.

Unlike traditional "spaghetti" controllers where all logic (jumping, sliding, crouching) is stuffed into a massive Update() loop, this controller separates every action into its own distinct class.

Why use this?

  • üß† Clean Architecture: Logic is separated into specific State classes.
  • üß© Modular: Easily add or remove capabilities without breaking the whole controller.
  • ‚ú® Polish: Built-in support for camera bobbing, tilts, and FOV changes.
  • üåä Physics: Includes sliding, crouching, and even swimming states out of the box.

Key Features

Feature Description
State Machine Robust hierarchical state machine handling transitions.
Movement Walk, Sprint, Crouch, Jump, Slide, Swim.
Camera Head bob, Strafe Tilt, Dynamic FOV zoom.

Understanding the core components will help you extend the controller effectively.

Core Components

1. FirstPersonController (The Context)

This is the main Monobehaviour script attached to your player. It holds:

  • References (Camera, CharacterController)
  • Configuration (Walk Speed, Jump Height)
  • Shared Data (IsGrounded, CurrentVelocity)

It delegates logic to the active State.

2. PlayerBaseState

The abstract base class for all states. It enforces the structure:

  • EnterState() - Called when state starts
  • UpdateState() - Called every frame
  • ExitState() - Called when leaving state
  • CheckSwitchStates() - Checks if we should transition

3. PlayerStateFactory

A central hub that creates and holds instances of your states. This prevents garbage collection spikes by reusing state instances.

‚ûï How to Add a New State

Want to add a Dash, Wall Run, or Jetpack ability? Follow this guide.

Step 1: Create the Source File

1 Create a new script named PlayerDashState.cs inheriting from PlayerBaseState.

using UnityEngine;
namespace EasyPeasyFirstPersonController 
{
    public class PlayerDashState : PlayerBaseState
    {
        public PlayerDashState(FirstPersonController currentContext, PlayerStateFactory playerStateFactory)
            : base(currentContext, playerStateFactory) { }

        public override void EnterState() 
        {
            // Start Dash Effect
            ctx.fovChangeSpeed = 20f;
            ctx.targetFov = 90f;
            Debug.Log("DASH START!");
        }

        public override void UpdateState() 
        {
            CheckSwitchStates();
            // Dash physics logic...
            ctx.characterController.Move(ctx.transform.forward * 20f * Time.deltaTime);
        }

        public override void ExitState() 
        {
             // Reset FOV
             ctx.targetFov = ctx.normalFov;
        }

        public override void CheckSwitchStates() 
        {
            // Return to ground after time
            if (dashTimer > 0.5f) 
            {
                SwitchState(factory.Grounded());
            }
        }
    }
}

Step 2: Register in Factory

2 Open PlayerStateFactory.cs and add a helper method.

public class PlayerStateFactory
{
    // ... existing states ...

    // Add this line:
    public PlayerBaseState Dash() => new PlayerDashState(context, this);
}

Step 3: Define Transitions

3 Go to an existing state (like PlayerGroundedState.cs) and add the condition to switch to your new state.

public override void CheckSwitchStates()
{
    if (ctx.input.jump && ctx.isGrounded)
    {
        SwitchState(factory.Jumping());
    }
    // ADD THIS:
    else if (Input.GetKeyDown(KeyCode.E)) 
    {
        SwitchState(factory.Dash());
    }
}
üí° Quick Tip: Always make sure to define the Exit Condition in your new state, otherwise the player will be stuck dashing forever!

Context Variables (ctx)

These variables are available in every State via the ctx variable.

Variable Type Description
characterController CharacterController Reference to the Unity component.
input InputManagerOld Access inputs like input.moveInput or input.jump.
isGrounded bool Updated every frame. True if standing on ground.
moveDirection Vector3 Current movement vector applied to the controller.
targetFov float Set this to smoothly interpolate the camera FOV.
